# Setting up design context

This post cover my initial prompts to create a south indian theme based wireframes for a tic tac toe gaming application. I used google stitch as my AI assistant for achieve this.

I start with a project overview, then dive in directly in design prompting.

## Project Overview

A premium, culturally-rooted Tic Tac Toe mobile game featuring high-fidelity, intricate line-art iconography based on South Indian temple architecture and traditional motifs.

## Master Screen: Contexting for core UI elements to be resused in subsequent screens.
### Prompt:
Act as an expert UI designer engineer. Generate a wireframe for the below idea.
Wireframe Idea: Simple structural skeleton design.

Design a wireframe with the following instructions-
1. The layout of the screen should be 9:12 for mobile apps 
2. Primary background of wireframe- Use the design colour profile of#0c5021 (deep forest green with slight yellow undertone). Maintain this pallette through the entirety of wifeframe without gradient flush
3. Add a tic tac toe 3x3 grid at the center of the wireframe with the below attributes-Grid Lines should be in Muted Gold ( #C0C0C0 at 40-50% opacity) grid lines should be open and not boxed or confined
4. Below the grid vertically distant ~40-60px from the grid position "KHEL TIC TAC" in all caps, #C9A227 colour and Noto Serif font semi-bolded. Align it in the centre
5. Do not add anything else on the wireframe not mentioned. no additional lines, headers, taskbars required

Ask any questions for clarity in making the wireframe.
