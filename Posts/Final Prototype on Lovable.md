# Final Prototype on Lovable
The struggles with generative is real. Generative AI will augment UI designers, not replace them, because design is not just interface creation—it’s problem framing, user empathy, system thinking, and decision-making under ambiguity.

However, it did help in creating a working MVP that conceptualized an idea to execution without dependancy. I used `Lovable` to upload my design spec to create a working MVP and used `Claude` to create PRD functional and feature requirement. 

# Product Requirements Documentation
Since my moat for this project was mainly designs, I needed a quick product requirements document that handled basic functioning. Designing a user journey from splash screen to game mode in a few clicks and peripheral journey that includes going to settings, or chat mode or checking game was all that was required.

Claude helped in creating this PRD within seconds. I created a basic V1 version for IOS and focused on monetization to be targeted on Ads. 

<b>Why Tic Tac Toe?-</b> This game in the casual games category ranks as the top 10 most searched apps.

<b>Goals-</b> I kept the P0, P1 and P2 goals to measure the success of the app quite simple.

<b>P1 — Must Have (Measurable)</b>
Time-to-game: User reaches an active game screen in 3 taps or fewer from app open (returning user) or 4 taps or fewer (first-time user).
Onboarding completion rate: 90%+ of first-time users complete name entry and reach the homepage.
D1 retention: 40%+ of users return to play within 24 hours of first install.
Game mode accessibility: All 4 game modes (Human vs AI, Human vs Human, Self Player, Play with Friends) are accessible from the Game Mode screen without navigating away.

<b>P2 — Should Have (Measurable)</b>
Theme engagement: At least 50% of active users switch themes at least once within their first 7 sessions.
Ad-Free upgrade conversion: 5%+ of monthly active users upgrade to Go Ad-Free within the first 3 months of launch.
Leaderboard participation: 30%+ of users check the Warriors / Royals Podium at least once per week.

<b>P3 — Immeasurable (Feelings)</b>
Users should feel a sense of cultural pride and delight when interacting with the app's Indian visual themes.
The game should feel effortless — like picking up a pen and playing on paper, just more beautiful.
Winning should feel celebratory; losing should feel motivating, not punishing.

## Some of my key learning lessons

1. AI is an assistive layer, not a decision-maker
Treat it as a co-pilot for ideation and speed, while humans own judgment, context, and final decisions.

2. Non-deterministic outputs require iteration
Tools like ChatGPT or Gemini produce variable outputs (randomness/temperature), so consistency comes from prompt refinement and constraints, not one-shot generation.

3. Prompting is an optimization loop
Effective use of AI involves iterative prompting, feedback loops, and constraint tuning to converge toward desired output.

4. Strong inputs → better outputs
Providing PRDs, design specs, user personas, and constraints significantly improves relevance and reduces ambiguity.

5. AI accelerates prototyping, not validation
It helps build MVPs and mockups faster, but user testing and validation remain human-driven.

6. Human oversight ensures coherence
AI can generate isolated screens, but maintaining design systems, consistency, and accessibility requires structured human review.

7. Context injection improves quality
Adding business goals, edge cases, and user scenarios leads to more usable and production-ready outputs.

8. Bias and hallucination need guardrails
Outputs must be validated against real data and requirements to avoid incorrect assumptions.

9. AI shifts designers from execution to orchestration
The role evolves toward problem framing, system thinking, and cross-functional alignment.

10. Efficiency gains come from workflows, not tools alone
Maximum impact comes when AI is integrated into repeatable workflows (design → iterate → test → refine), not used ad hoc.
